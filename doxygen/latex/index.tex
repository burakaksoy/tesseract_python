{\ttfamily tesseract\+\_\+python} contains Python wrappers for the Tesseract robot motion planner, generated using SWIG. These wrappers contain most of the Tesseract functionality, including scene loading and management (URDF,SRDF, meshes), collision checking (Bullet, FCL), kinematics (KDL, OPW, UR), planning (OMPL, Descartes, Traj\+Opt), and visualization (tesseract\+\_\+viewer\+\_\+python)

Standalone packages are provided on Py\+Pi (pip install) for Windows and Linux, containing all the native dependencies for Python 3.\+7+.

{\bfseries{Note that these are low level wrappers.}} The lifecycle of objects follow the underlying C++ objects, meaning that the target of C++ references may be destroyed before the reference, leading to a memory error. These wrappers do not attempt to change the memory lifecycle of the underlying C++ objects.\hypertarget{index_autotoc_md1}{}\doxysection{Installation}\label{index_autotoc_md1}
Standalone packages are provided on Py\+Pi (pip install) for Windows and Linux, containing Tesseract, Tesseract Planning, andall the native dependencies for Python 3.\+7+. These packages have been tested on Windows 10, Ubuntu 20.\+04, and Ubuntu 22.\+04, but should work on any relatively recent x64 Windows or Linux operating system. Packages are available for Python 3.\+7 -\/ 3.\+10.

\hypertarget{index_autotoc_md2}{}\doxysection{Example}\label{index_autotoc_md2}
ABB Tesseract viewer plan and viewer example\+:

Install {\ttfamily tesseract\+\_\+robotics} and {\ttfamily \mbox{\hyperlink{namespacetesseract__robotics__viewer}{tesseract\+\_\+robotics\+\_\+viewer}}} as shown in Installation section.

Clone {\ttfamily tesseract} and {\ttfamily tesseract\+\_\+python} repositories to retrieve example assets. This is not necessary if the example assets are not used.


\begin{DoxyCode}{0}
\DoxyCodeLine{git clone -\/-\/depth=1 https://github.com/tesseract-\/robotics/tesseract.git}
\DoxyCodeLine{git clone -\/-\/depth=1 https://github.com/tesseract-\/robotics/tesseract\_python.git}

\end{DoxyCode}


Set the {\ttfamily TESSERACT\+\_\+\+SUPPORT\+\_\+\+DIR} so the example can find the URDF resources\+:

Linux\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{export TESSERACT\_SUPPORT\_DIR=`pwd`/tesseract/tesseract\_support}

\end{DoxyCode}


Windows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{set TESSERACT\_SUPPORT\_DIR=\%CD\%/tesseract/tesseract\_support}

\end{DoxyCode}


Now run the example!

Windows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{cd tesseract\_python\(\backslash\)tesseract\_viewer\_python\(\backslash\)examples}
\DoxyCodeLine{python abb\_irb2400\_viewer.py}

\end{DoxyCode}


Linux\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{cd tesseract\_python/tesseract\_viewer\_python/examples}
\DoxyCodeLine{python3 abb\_irb2400\_viewer.py}

\end{DoxyCode}


And point a modern browser to {\ttfamily \href{http://localhost:8000}{\texttt{ http\+://localhost\+:8000}}} to see the animation!

\hypertarget{index_autotoc_md3}{}\doxysection{Tesseract Python Supported Packages}\label{index_autotoc_md3}

\begin{DoxyItemize}
\item {\bfseries{tesseract\+\_\+collision}} – This package contains privides a common interface for collision checking prividing several implementation of a Bullet collision library and FCL collision library. It includes both continuous and discrete collision checking for convex-\/convex, convex-\/concave and concave-\/concave shapes.
\item {\bfseries{tesseract\+\_\+common}} – This package contains common functionality needed by the majority of the packages.
\item {\bfseries{tesseract\+\_\+environment}} – This package contains the Tesseract Environment which provides functionality to add,remove,move and modify links and joint. It also manages adding object to the contact managers and provides the ability.
\item {\bfseries{tesseract\+\_\+geometry}} – This package contains geometry types used by Tesseract including primitive shapes, mesh, convex hull mesh, octomap and signed distance field.
\item {\bfseries{tesseract\+\_\+kinematics}} – This package contains a common interface for Forward and Inverse kinematics for Chain, Tree\textquotesingle{}s and Graphs including implementation using KDL and OPW Kinematics.
\item {\bfseries{tesseract\+\_\+scene\+\_\+graph}} – This package contains the scene graph which is the data structure used to manage the connectivity of objects in the environment. It inherits from boost graph and provides addition functionality for adding,removing and modifying Links and Joints along with search implementation.
\item {\bfseries{tesseract\+\_\+support}} – This package contains support data used for unit tests and examples throughout Tesseract.
\item {\bfseries{tesseract\+\_\+urdf}} -\/ This package contains a custom urdf parser supporting addition shapes and features currently not supported by urdfdom.
\item {\bfseries{tesseract\+\_\+visualization}} – This package contains visualization utilities and libraries.
\item {\bfseries{tesseract\+\_\+command\+\_\+language}} -\/ This is a generic programing language used as input for motion and process planning. It is very similar to how you currently program a robot on an industrial teach pendant.
\item {\bfseries{tesseract\+\_\+motion\+\_\+planners}} – This package contains a common interface for Planners and includes implementation for OMPL, Traj\+Opt, Traj\+Opt IFOPT and Descartes.
\item {\bfseries{tesseract\+\_\+process\+\_\+managers}} – This package contains a common interface for Process Planning and includes implementation for a wide variaty of process found industrial automation like painting, griding, welding, pick and place and more.
\item {\bfseries{tesseract\+\_\+time\+\_\+parameterization}} – This package contains a time parameterization algorithms and includes iteritive spline.
